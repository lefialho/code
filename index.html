Esse teste foi importante para estudar alguns comandos do git, tais como:
  • git status que serve para exibir o status do seu repositório atual.
  • git add para preparar o arquivo para o seu versionamento que funciona como uma área do git, no qual os arquivos possam ser enviados ao Github. ADD não está adicionando um arquivo novo ao repositório, mas sim dizendo que o arquivo (sendo novo ou não) está sendo preparado para entrar na próxima revisão do repositório.
  • git commit -m cria uma revisão com um número e um comentário, que será vista por todos.
  • git push, usado para trazer todas as modificações que estão no github para o projeto local.
  • git pull é usado para publicar todos os commits para o github.
  • git checkout utilizado para trocar de branch.
